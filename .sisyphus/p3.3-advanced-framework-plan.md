# Phase 3.3: Advanced Framework Components - Implementation Plan

> **Version**: 1.0  
> **Date**: 2026-04-11  
> **Status**: 📋 Draft  
> **Phase**: P3.3 (Foundation → Framework)  
> **Estimated Duration**: 3-4 days  
> **Complexity**: High (Framework infrastructure)

---

## Executive Summary

**Goal**: Transition from "hardcoded modules" to a "configurable, framework-driven system" by implementing four critical advanced framework components:

1. **ModuleRegistry** — JSON-driven module instantiation and topology management
2. **StreamAdapter** — Template-based TLM ↔ RTL bundle bridging
3. **FragmentMapper** — Protocol translator for fragmentation/reassembly
4. **PortPair+** — Enhanced connection abstraction with type validation

**Current State**: Phase 3.2 completed `TLMModule` base class with transaction lifecycle hooks. `CacheV2`, `MemoryV2`, `CrossbarV2` now inherit from `TLMModule`. Module instantiation exists in `ModuleFactory::instantiateAll()` but requires hardcoded JSON paths.

**Target State**: Fully declarative topology configuration, type-safe module connections, and seamless TLM↔RTL data flow bridging.

### 1.1 ModuleRegistry

**Purpose**: Centralized registry for declarative module instantiation from JSON topology files.

**Current State**:
- `ModuleFactory` exists (`include/core/module_factory.hh`)
- `instantiateAll(const json& config)` method already parses JSON and creates instances
- `REGISTER_MODULE` macro exists for module registration
- **Gap**: No centralized registry API, no schema validation, no runtime discovery

**Target Design**:

```cpp
// include/framework/module_registry.hh (NEW)
class ModuleRegistry {
public:
    // Singleton access
    static ModuleRegistry& instance();
    
    // Registration (automatic via REGISTER_MODULE macro)
    template<typename T>
    void register_type(const std::string& name);
    
    // Instantiation from JSON
    SimObject* create_module(const std::string& name, 
                             const std::string& type,
                             const json& params,
                             EventQueue* eq);
    
    // Topology loading
    void load_topology(const std::string& json_path);
    
    // Discovery
    std::vector<std::string> get_registered_types() const;
    bool has_type(const std::string& type) const;
    
    // Instance management
    SimObject* get_instance(const std::string& name) const;
    std::unordered_map<std::string, SimObject*> get_all_instances() const;
    
private:
    std::unordered_map<std::string, CreateSimObjectFunc> registry_;
    std::unordered_map<std::string, SimObject*> instances_;
};

// Macro for automatic registration
#define REGISTER_MODULE(T, NAME) \
    namespace { \
        struct NAME##_Registrar { \
            NAME##_Registrar() { \
                ModuleRegistry::instance().register_type<T>(NAME); \
            } \
        }; \
        static NAME##_Registrar g_##NAME##_reg; \
    }
```

### 1.2 StreamAdapter

**Purpose**: Template-based mechanism to bridge `Packet`-based TLM flows with `Bundle`-based RTL flows.

**Current State**:
- `SystemcAdapter` exists (`include/sc_core/sc_adapter.hh`) but is SystemC-specific
- `TLMBundle` structure defined in architecture docs but not implemented
- No generic template adapter for CppHDL streams

**Target Design**:

```cpp
// include/framework/stream_adapter.hh (NEW)

// =========== Bundle Type ===========
struct TLMBundle {
    uint64_t transaction_id;  // From TransactionContext
    uint64_t flow_id;         // Optional, for QoS
    uint64_t address;
    uint64_t length;
    uint8_t  data[64];        // Max 64-byte payload
    uint8_t  strb;            // Byte enable
    bool     write;           // Write flag
    uint8_t  priority;
    
    // Serialization helpers
    size_t serialized_size() const { return sizeof(*this); }
};

// =========== Adapter Traits ===========
template<typename BundleT>
struct BundleTraits {
    using bundle_type = BundleT;
    using value_type = typename BundleT::value_type;
    
    static constexpr size_t data_width = sizeof(BundleT) * 8;
    static constexpr bool has_transaction_id = requires(BundleT b) { b.transaction_id; };
};

// =========== StreamBridge (TLM ↔ ch_stream<Bundle>) ===========
template<typename BundleT>
class StreamBridge {
public:
    StreamBridge(const std::string& name, EventQueue* eq);
    
    // TLM → Stream direction
    void send_packet(Packet* pkt);
    
    // Stream → TLM direction
    void handle_bundle(const BundleT& bundle);
    
    // Register callbacks
    using BundleHandler = std::function<void(const BundleT&)>;
    void set_bundle_handler(BundleHandler handler);
    
private:
    std::string name_;
    EventQueue* eq_;
    BundleHandler bundle_handler_;
    
    // TLM → Bundle conversion
    BundleT packet_to_bundle(Packet* pkt);
    Packet* bundle_to_packet(const BundleT& bundle);
};

// =========== FlowBridge (TLM ↔ ch_flow<Bundle>) ===========
template<typename BundleT>
class FlowBridge {
    // Similar to StreamBridge but for unidirectional flows
    // Ch implementation depends on CppHDL API
};
```

### 1.3 FragmentMapper

**Purpose**: Protocol translator for handling fragmentation/reassembly logic defined in `FragmentBuffer`.

**Current State**:
- `FragmentBuffer` struct exists in `sim_object.hh` and `tlm_module.hh`
- Fragment reassembly logic partially implemented in `TLMModule::processWithFragmentation()`
- `TransactionContextExt` has `fragment_id`, `fragment_total` fields
- **Gap**: No dedicated mapper class, no protocol-specific implementations

**Target Design**:

```cpp
// include/framework/fragment_mapper.hh (NEW)

// =========== Fragment Concept ===========
template<typename PayloadT>
struct Fragment {
    PayloadT data;
    uint8_t fragment_id;
    uint8_t fragment_total;
    bool is_first;
    bool is_last;
    uint64_t transaction_id;
    
    bool is_complete() const { return is_last; }
};

// =========== FragmentMapper Base ===========
template<typename BundleT>
class FragmentMapper {
public:
    FragmentMapper(const std::string& name, EventQueue* eq, size_t max_fragments = 256);
    
    // Fragmentation: Bundle → Fragment stream
    std::vector<Fragment<BundleT>> fragment_bundle(const BundleT& bundle, 
                                                    size_t max_fragment_size);
    
    // Reassembly: Fragment stream → Bundle
    std::optional<BundleT> reassemble_fragment(const Fragment<BundleT>& frag);
    
    // Status query
    bool is_reassembly_complete(uint64_t transaction_id) const;
    size_t get_pending_fragments(uint64_t transaction_id) const;
    
protected:
    std::string name_;
    EventQueue* eq_;
    size_t max_fragments_;
    
    // Reassembly buffers: transaction_id → {fragment_id → fragment}
    std::unordered_map<uint64_t, 
        std::map<uint8_t, Fragment<BundleT>>> reassembly_buffers_;
    
    virtual void on_reassembly_complete(uint64_t tid, const BundleT& bundle);
};

// =========== Protocol-Specific Mappers ===========

// AXI4 Mapper (burst fragmentation)
template<typename BundleT>
class AXI4Mapper : public FragmentMapper<BundleT> {
public:
    AXI4Mapper(const std::string& name, EventQueue* eq, size_t burst_size = 16);
    
    // AXI4-specific: Calculate beats from burst length
    size_t calculate_beats(uint64_t address, size_t length) const;
    
protected:
    size_t burst_size_;
};

// CHI Mapper (transaction-based fragmentation)
template<typename BundleT>
class CHIMapper : public FragmentMapper<BundleT> {
    // CHI-specific implementation
};

// TileLink Mapper (channel-based fragmentation)
template<typename BundleT>
class TileLinkMapper : public FragmentMapper<BundleT> {
    // TileLink-specific implementation
};
```

**File Location**: `include/framework/fragment_mapper.hh`

**Dependencies**:
- Depends on `FragmentBuffer` concepts from `sim_object.hh`
- Depends on `TransactionContextExt` for fragment metadata
- **Does NOT depend on StreamAdapter** — orthogonal concerns (can be composed)

### 1.4 PortPair Enhancements

**Purpose**: Robust connection abstraction with type validation and connectivity checking.

**Current State**:
- `PortPair` is a simple struct (`include/core/simple_port.hh`):
  ```cpp
  class PortPair {
  public:
      SimplePort* side_a;
      SimplePort* side_b;
      PortPair(SimplePort* a, SimplePort* b);
  };
  ```
- **Gap**: No validation, no type checking, no connection metadata

**Target Design**:

```cpp
// include/core/port_pair.hh (enhanced, rename from simple_port.hh)

// =========== Connection Metadata ===========
struct ConnectionInfo {
    std::string src_module;
    std::string src_port;
    std::string dst_module;
    std::string dst_port;
    int latency_cycles = 0;
    size_t input_buffer_size = 0;
    size_t output_buffer_size = 0;
    std::vector<uint8_t> vc_priorities;
};

// =========== Port Type Tag ===========
enum class PortType {
    TLM_INITIATOR,    // Sends requests
    TLM_TARGET,       // Receives requests
    RTL_INPUT,        // Receives bundles
    RTL_OUTPUT,       // Sends bundles
    BIDIRECTIONAL     // Both directions
};

// =========== Enhanced PortPair ===========
class PortPair {
public:
    SimplePort* side_a;
    SimplePort* side_b;
    ConnectionInfo info;
    PortType type_a;
    PortType type_b;
    bool validated = false;
    
    PortPair(SimplePort* a, SimplePort* b, 
             PortType ta = PortType::BIDIRECTIONAL,
             PortType tb = PortType::BIDIRECTIONAL);
    
    // Validation
    bool validate_compatibility() const;
    bool is_valid() const { return validated; }
    
    // Connection statistics
    uint64_t packets_transferred = 0;
    uint64_t bytes_transferred = 0;
    uint64_t contention_cycles = 0;
    
    // Lifecycle
    void disconnect();
    bool is_connected() const;
};

// =========== Connection Validator ===========
class ConnectionValidator {
public:
    static bool validate_port_compatibility(const SimplePort* a, const SimplePort* b);
    static bool validate_connection_info(const ConnectionInfo& info);
    static std::string get_compatibility_error(const SimplePort* a, const SimplePort* b);
};
```

**File Location**: `include/core/port_pair.hh` (enhance existing `simple_port.hh`)

**Dependencies**:
- Depends on `SimplePort` (existing)
- No new dependencies
