# PRD-001: CppTLM 多层次混合仿真平台

> **版本**: 1.0  
> **日期**: 2026-04-09  
> **状态**: Draft  
> **优先级**: P0

---

## 1. 产品愿景

**一次建模，多粒度仿真** — 在同一 C++ 框架下，支持事务级 (TLM) 和周期精确级 (RTL) 建模的无缝混合仿真。

---

## 2. 目标用户

| 用户类型 | 需求场景 |
|---------|---------|
| **芯片架构师** | 早期架构探索，快速评估不同设计选项的性能影响 |
| **RTL 设计师** | 在系统级上下文中验证 RTL 模块，无需等待完整 SoC 集成 |
| **软件开发人员** | 在 RTL 完成前开始软件开发，使用 TLM 模型 |
| **验证工程师** | 混合仿真环境下的功能验证和性能分析 |

---

## 3. 核心需求

### 3.1 功能性需求

| ID | 需求描述 | 优先级 |
|----|---------|--------|
| **FR-001** | 支持 TLM 模块和 RTL 模块在同一仿真中混合运行 | P0 |
| **FR-002** | 交易上下文端到端追踪（transaction_id 穿透所有模块） | P0 |
| **FR-003** | 支持多种总线协议（AXI4, CHI, TileLink） | P1 |
| **FR-004** | 支持模块级实现选择（tlm/rtl/compare/shadow 模式） | P1 |
| **FR-005** | 性能分析报告，带置信度评分 | P2 |
| **FR-006** | VCD 波形导出，带 transaction_id 注释 | P2 |

### 3.2 非功能性需求

| ID | 需求描述 | 验收标准 |
|----|---------|---------|
| **NFR-001** | 向后兼容 | 现有 TLM 代码零修改 |
| **NFR-002** | 性能开销 | 混合仿真相比纯 TLM 开销 < 10% |
| **NFR-003** | 仿真速度 | 纯 TLM > 1000 KIPS |
| **NFR-004** | 代码质量 | 零编译警告，单元测试覆盖率 > 90% |

---

## 4. 用户故事

### US-001: 架构师快速探索

**作为** 芯片架构师  
**我希望** 能快速搭建不同 NoC 拓扑的仿真环境  
**以便** 在 RTL 开发前评估带宽和延迟

**验收标准**:
- [ ] JSON 配置定义拓扑
- [ ] 5 分钟内完成新拓扑搭建
- [ ] 延迟预测误差 < 20%

### US-002: RTL 设计师混合验证

**作为** RTL 设计师  
**我希望** 将我的 RTL 模块放入 TLM 系统中仿真  
**以便** 在系统级上下文中验证功能

**验收标准**:
- [ ] RTL 模块可通过适配器接入
- [ ] transaction_id 端到端追踪
- [ ] compare 模式验证功能等价性

### US-003: 软件开发者提前开发

**作为** 软件开发人员  
**我希望** 在 RTL 完成前使用 TLM 模型开发驱动  
**以便** 软硬件并行开发

**验收标准**:
- [ ] TLM 模型行为与 RTL 一致
- [ ] 支持标准寄存器接口
- [ ] 可无缝切换到 RTL

## 5. 约束与依赖

### 5.1 技术约束

| 约束 | 说明 |
|------|------|
| **C++ 标准** | C++17（兼容 SystemC 2.3.4） |
| **SystemC** | 可选依赖，通过 USE_SYSTEMC 编译选项控制 |
| **CppHDL** | 外部依赖，位于 /workspace/CppHDL |
| **构建系统** | CMake，支持 Ninja/Make 生成器 |

### 5.2 外部依赖

| 依赖 | 版本 | 用途 |
|------|------|------|
| SystemC | 2.3.4 | TLM 2.0 接口（可选） |
| CppHDL | latest | RTL 建模框架 |
| nlohmann/json | 3.x | JSON 配置解析 |
| Catch2 | 2.x | 单元测试框架 |

---

## 6. 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| **性能开销超预期** | 中 | 高 | 非关键路径 TLM Bypass 模式 |
| **CppHDL 接口变更** | 中 | 中 | 锁定 commit hash，编写接口测试 |
| **transaction_id 耗尽** | 低 | 中 | 64 位 ID + 回收机制 |
| **CDC 时序缺陷** | 中 | 高 | 异步 FIFO + 形式化验证 |

---

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
